D

Description

给定一个长度为 的序列 和 ，求满足出现次数最多的数的出现次数小于 ​ 的连续子序列的数目。

Solution

由于是连续的子序列，而且发现选的数越多越可能满足，考虑双指针。

对于每个 ，我们找到最小的能满足这个条件的 ​，然后后面的就随便选了，把它对答案的贡献加进去，就完事了。

至于怎么找 呢，我们用一个 map 存每个数的出现次数就行了。

E

Description

给定一个点数为 的完全图，从中删除 条边，求得到的图的联通块。。

Solution

发现一个事情：相对于一个很大的完全图，删除的边数实在少得可怜。量化一下发现最多只能让约 ​​​ 个点成为一个独立的联通块。

这实在是小的可怜。也就是说大部分点实际上是连通的。

考虑到当能删完的时候，，那么我们直接找一个在删除的边中出现次数最小的点，把与它相连的点全都取出来当成一个联通块。

剩下的点爆搜一下连通块就行。我的实现非常暴力而且恶心，使用了 map 复杂度大概是 ​​。最大的点跑得很慢。

事实上我不知道我这个复杂度到底对不对，反正是卡常卡过去了...

看到有用并查集的做法能做到 ​，值得一学。。。
