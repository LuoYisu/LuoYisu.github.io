概述

今天是构造，题似乎不是很难，然而我挂的很惨。

期望：​，实际 ​。大家都挂的很惨，于是我 RK13/23。。。

前言

实际上我从来没有做过构造题，唯一的是前一天听同学口胡的一道 CF ​ 分的题，于是今天十分没底。

赛时

看到保证 T1 是最简单的题，我立马定好策略：切掉 T1 再说。（T1：CF1439A2）

于是开始想，大概想了十分钟吧，思路就已经成型了：枚举每个 ​​ 正方形左上角的点，从第一行扫到倒数第三行，扫的时候保证该行所有 都已经变成 ，其他随意。接着横着扫倒数第二行，一直扫到倒数第三列。这个时候除了右下角 的四个数，所有数都已经变为 了。而对于最后四个数，可能的剩余未熄灭灯数只有 ​​ 种。直接用打表/分类讨论之类的方法搞出来就行。

Code

写完 T1，按传统规矩自然是 DEBUG 了好久，确定无误之后已经过去一个半小时了，感觉自己得了 分，一阵狂喜，赶快开下面的题。

接着看后面的题，决定先打暴力。

首先打了个 T3 的 分暴力，一阵狂喜，然后去写 T4，分析了一下发现我似乎能切。于是开始写，但是调到心态爆炸，于是在剩余 ​​ 的时候勉强调完了。

T2 急的一笔，快速打了个暴力，希望能得 分。不过也没测试，草草收场了。

然后快结束的时候有人告诉我 T1 的文件名写错了，我：？？？

结果还真写错了，我连忙改文件名和 freopen 里的文件名。

时间到了，我一看，我 freopen 里的文件名没改上，我：？？？发生什么事了？？？

行吧， 分没了。

赛后

我一看分数就傻眼了。

再一看：T3 File Error：“未找到选手输出文件”

哦，仔细一看，我把一个 freopen 的文件名写错了，好家伙，这又少了 分。

T4 果然挂了，不出所料，不过我和 Solution 的思路是差不多的，但是实现似乎有点问题。

又在 OJ 上交了一发，好家伙，T1 直接 AC 了，T3 分了。

忽然意识到，我因为 freopen 痛失了 分。太伤了，加上这些分我似乎能进 Rank 5。

题目都不是很难，T2 其实是个分类讨论题，感觉再来点时间也能想出来，倒是 T3 似乎思维难度最大，有点类似染色（

总结

狂喜把自己狂喜没了，这次在做题上还算是比较完美的，不过下一次必须认真比对文件名之后（或者直接复制）再去搞下一道题。

这次太着急了，所以出了一堆错误。

T1 明明几分钟就想出来正解了，结果后面又否定自己，最后又发现自己是对的，也浪费了大量时间。

代码实现的时候也浪费了时间。

只能说码力还是不够，正解和暴力都写得太慢了，结果没有时间支持我深入思考每道题，相信随着刷题数的增多我这个能力能增强的。
