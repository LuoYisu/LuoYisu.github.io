写一下这个题的题解吧。

Description

给出一个有向图，每个边有一个边权，当你从一个点出发，经过每条边的时间为边权乘上 ，其中 为这条边是第几条被经过的边。

你需要求出在这个图上能行走的最长时间，以及这样的路径数目（如果答案是无限大，输出 inf）。

。

Solution

首先假设 ， 的情况一会再考虑。

显然，如果有一个有正权的环，我们输出两个无限大就好了，因此我们缩点，下文考虑一个 DAG 的情况。

我们直接把最长时间一位一位考虑，因为全都非零，位数越多越大。

dp 出一个最长路，然后倒着推回去即可。

因为最大位到最低位是一个贪心的过程，我们把所有最长路放到一个 std::set 里，然后每次找集合中能在任意一条最长路上的最大边权的边走，递归下去就能找到最长时间，然后再递推一遍路径数就好了。

接下来考虑存在 的情况。

先考虑 环，如果 环能走到正权边，那就是两个 inf，如果 环在最长路的结尾（或之后），那么方案数就是 inf。

如果边权全部是 ，那么最长时间就是 ，在这种情况下，如果存在 环，那么路径数是 inf，否则路径数是这个 dag 上路径总数。

这样我们就去除了 环的影响。

然后我们需要考虑最大时间先导零的问题，首先 dp 的时候要额外记录一个当前最高位非零的最长路，以这个的最大值作为位数。

因为有先导零，我们需要把所有终点能到达的点也设为终点（因为都是走 边），然后和之前一样推回去就好了。
